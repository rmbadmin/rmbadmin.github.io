这篇文章可能没用，我又卡在Emulation-HW的编译（具体是链接）这一步了……问了配过的学长，他给了这么一篇参考博客，最近有别的项目所以还没来得及实验，各位可以试一试，有效或者无效欢迎留个评论，帮助更多的人少走些弯路……

系统配置

硬件：KCU 1500 插在 PCIe 3.0 x16 上。

软件：操作系统版本使用 Ubuntu 16.04.4。这里注意，一定是 16.04.4，不要使用更新或更老的版本，16.04.6 是不可以用的，而且使用 14.04.4 之后也不要更新系统（ apt update ），否则会卡在安装内核驱动那一步。

软件下载

只需要下载 SDAccel 2018.2 （在线安装包或离线包）即可，不需要下载下面那个 KCU 1500 Platform。2018.2 的下载是放在 Archive 里的，点开就能找到。

安装步骤

参考这篇文档，全部照做即可。

[已废弃]【小白向】Xilinx SDAccel 安装学习中遇到的坑和解决办法

发表于 2019-05-13 更新于 2019-09-21 分类于 水的一笔

一切的一切都从这个万恶的网页开始…… 左边一栏第一个下载，和第二个教程。软件版本是 2018.3。 1. 不使用代理，在线安装包下载速度极慢，我这里大概要下两百多小时。 2. 离线安装包 22G，安装完总大小大概占 65G 以上，虚拟机留出足够空间。 3. 离线安装包 22G，直接复制到虚拟机里可能导致虚拟机系统崩溃（vbox + ubuntu，原因未知），解决方案：共享文件夹，如有必要 su 切换到 root 权限进行复制，sudo 复制也会导致系统崩溃（原因未知）。 4. 解压后发现这是个 Windows 的离线包，Linux 用不了。 5. Windows里安装离线包后发现没有教程对应的设备，可以参考这个网页的解决方案。 6. 这里的软件包仅有 Red Hat/CentOS 版本和 Ubuntu 版本，也就是说 Windows 版离线包安装的程序是无法使用的。需要使用在线安装包安装 Linux 版，记得使用代理。 7. 装完了按教程做，到了下载模板和例子那一步（8. Select SDAccel Examples, and then click Download.），会出现网络错误的提示，让你检查连接或者检查代理设置。而实际原因可能是因为没有安装 git，安装一个就好了。 8. 例程的运行需要OpenCL，提前装好。 总结： 一定要使用在线安装包安装 Linux 版（系统一定要使用 Red Hat/CentOS 或者 Ubuntu 16/18）。如果很慢想办法开代理。然后提前安装好 git 和 OpenCL，不然出了错也不会报。

【小白向】 Tensorflow 在老处理器及奔腾处理器上安装报错“DLL load failed”的解决方案

发表于 2018-12-06 更新于 2019-09-21 分类于 水的一笔

表现：”DLL load failed: 动态链接库(DLL)初始化例程失败“ 首先：如果遇到这个错误，那么建议还是换一个CPU（或者换一台电脑），也许你的电脑性能已经不足以运行Tensorflow了，如果你不是打算在这台机器上严肃地跑程序，只是想试试代码是否能跑，而程序主要是跑在你另一台高性能的计算机上的话，那么可以接着看下去。 原因推测：CPU缺少某些指令集。 解决方案：安装老版本Tensorflow，我安装的是1.5版本的。 具体步骤： 首先卸载你安装的新版本Tensorflow pip uninstall tensorflow 然后安装你想装的版本 pip install tensorflow==1.5

【小白向】 Tensorflow 使用 GPU 加速中遇到的坑和解决方案

发表于 2018-10-17 更新于 2019-09-21 分类于 其它语言 ， 心得技巧 ， 水的一笔

1. python 版本不要过高。目前我使用的是 python 3.6.6，python 3.7是不能使用的。 2. 早期（两个月前） tensorflow 官网的文档说要安装 tensorflow 和 tensorflow-gpu 两个 pip 包，但是现在只需要安装一个了，如果同时安装了两个会使用 CPU 计算而不是用 GPU 。解决方案是卸载这两个包，重新 pip install tensorflow-gpu。 3. CUDA 版本不要过高，目前 CUDA 9.0 是可以使用的，更新的版本看 tensorflow 的 install 文档， cudnn 版本也要配套。 4. 第一次运行的时候 tensorflow 会添加 GPU ，这个过程会比较慢，请耐心等待…… 总结，大多数坑都是版本过新导致的，请严格按照 Tensorflow 的 Install 文档的要求版本安装其他程序，最新的版本未必被 Tensorflow 支持。 最后吐槽一句，显存不够用啊……GPU没怎么跑，显存先炸了……

NVIDIA安装程序无法继续。 需要重新启动系统。

发表于 2017-08-30 更新于 2019-09-21 分类于 日常 ， 水的一笔

在昨天晚上因为想更新显卡驱动，结果发现一直都没法装上，如下提示： 【因为我已经解决了问题装完了，所以这图是随便找到，版本号不要在意。。】 看到提示后我也重启过N遍电脑了，打开都是这个提示，然后我开始了卸载驱动。 没想到卸载驱动还是这个提示，不过在用了别的方法强制卸载后。发现还是一直提示重启。 没有办法完成安装。 最后发现是注册表里有个Nvidia_Reboot_Required来标记你的机子是否重启，一直没有被删除导致。 手动删除他就能安装驱动了。 1.WIN+R打开运行 2.键入 “regedit” 3.点开“HKEY_LOCAL_MACHINE\SOFTWARE” 4.在底下找到“Nvidia_Reboot_Required”（或类似的名称），并将其删除。 然后再打开你的nvidia安装程序，问题解决了~
