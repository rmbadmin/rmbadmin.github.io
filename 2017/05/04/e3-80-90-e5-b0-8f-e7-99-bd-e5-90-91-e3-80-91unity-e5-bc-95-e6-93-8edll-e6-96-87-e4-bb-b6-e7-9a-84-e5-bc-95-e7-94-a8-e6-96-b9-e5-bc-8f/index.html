【小白向】Unity引擎DLL文件的引用方式

发表于 2017-05-04 更新于 2019-09-21 分类于 Csharp ， 其它语言

最近需要在Unity引擎中使用网络通信，奈何直接引用.NET的TCP Client类似乎没什么效果，所以想到打包成DLL再调用，于是在网上找了如何在Unity中使用DLL文件，但是似乎都没什么用。刚刚在Unity的官方文档里找到了“官方”的解决方案，整理如下。 DLL**文件的生成** 关于编译器： 如果你要编译的模块不依赖与Unity的API，则选择合适的编译器编译即可，如果需要与Unity引擎交互，可以引用UnityEngine.dll和UnityEditor.dll这两个DLL文件。它们的位置为：

Windows：Program Files\Unity\Editor\Data\Managed\

macOS：Applications/Unity/Unity.app/Contents/Frameworks/Managed/

注：根据我遇到的情况，编译的DLL如果使用了.NET库，.NET版本不要高于3.5。 DLL**文件的使用** 将DLL文件拷贝至Unity引擎的Assets文件夹中，然后在C#脚本的开头处加入using。如，对与DLLTest.dll来说，应加入“using DLLTest;”。当将脚本添加到场景中的一个对象上后，点击播放即可发现脚本已经在工作了。 注：网上有很多说要放在什么Scripts文件夹或者Plugins文件夹下的，但是根据我的情况，放在那些地方都会导致编译器找不到dll，还是要放在根目录下。就是这个地方把我坑惨了…… 关于使用MonoDevelop调试，需要将mdb文件复制进Assets或Plugins文件夹。VS编译的DLL没有mdb文件，可以使用工具进行转换，位置如下：

Program Files\Unity\Editor\Data\Mono\lib\mono\2.0\pdb2mdb.exe

其命令行参数为pdb文件，会在同一目录下生成对应的mdb。 打开Tools菜单（Windows）或MonoDevelop-Unity menu (macOS)，调整设置Add-in Manager > Installed tab > Unity > select Mono Soft Debugger Support for Unity > Enable之后即可开始调试。
